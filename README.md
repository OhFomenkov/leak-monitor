# Sentinel Monitor (SOC Dashboard)

**Sentinel Monitor** — это система мониторинга утечек учетных данных в реальном времени. Проект предназначен для специалистов по информационной безопасности (SOC-аналитиков) с целью обнаружения скомпрометированных учетных записей сотрудников компании в открытых источниках.

## Основные возможности
- **Telegram Collector**: Мониторинг Telegram-каналов на наличие "сливов" (email:pass).
- **Web Scraper**: Мониторинг OSINT-источников для выявления новых дампов.
- **API Security Checks**: Интеграция с внешними авторитетными агрегаторами утечек (например, LeakCheck.io) для мгновенной проверки email-адресов на компрометацию.
- **Matching Engine**: Автоматическое сопоставление найденных утекших данных с реестром сотрудников компании.
- **SOC Dashboard**: Веб-интерфейс на Flask для управления инцидентами, конфигурацией источников и визуализации безопасности.

## Технологический стек
- **Язык**: Python 3.10+
- **Backend**: Flask
- **База данных**: PostgreSQL
- **Парсинг**: Telethon (Telegram API), BeautifulSoup4, Requests
- **UI**: Bootstrap 5, Jinja2

## Быстрый запуск

### 1. Подготовка
```bash
# Клонирование репозитория
git clone https://github.com/OhFomenkov/leak-monitor.git
cd leak-monitor

# Создание виртуального окружения
python -m venv venv
venv\Scripts\activate  # Для Windows
Если возникла ошибка, то предварительно используйте:
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope Process
# Установка зависимостей
pip install -r requirements.txt
```
### 2. Конфигурация
```bash
Создайте файл .env в корне проекта на основе .env.example и укажите необходимые данные:
Данные для подключения к PostgreSQL (DB_NAME, DB_USER, DB_PASSWORD, и т.д.)
Telegram API Credentials (BOT_TOKEN)
Секретный ключ (SECRET_KEY) - для подписи сессии(Cookies)
Для генерации секретного ключа используйте:
python -c "import secrets; print(secrets.token_hex(32))"
```
### 3. Инициализация базы данных
```bash
# Создание таблиц
python -m app.database.db_init

# Наполнение демо-данными для проверки
python scripts/db_seed.py
```
### 4. Запуск
```bash
# Запуск веб-интерфейса
python -m app.web.app
```
## Демонстрация веб-скрапера
```bash
Для тестирования системы без подключения к реальным источникам, вы можете использовать встроенный тестовый стенд:

1. **Запуск стенда**: Выполните `python dev_tools/forum_mock.py` (имитация форума запустится на `http://127.0.0.1:5001`).
2. **Проверка стенда**: Откройте `http://127.0.0.1:5001` в браузере, чтобы убедиться, что он работает.
3. **Настройка**: В основном веб-интерфейсе (на порту 5000) перейдите в раздел **Настройки** -> **Источники веб-скрапера**. Нажмите "Добавить URL" и введите:
    - **Название**: `Тестовый стенд`
    - **URL**: `http://127.0.0.1:5001`
4. **Имитация данных**: На тестовом стенде (на порту 5001) добавьте запись с учетными данными в формате `email:password` (в зависимости от вашей реализации стенда).
5. **Запуск**: В веб-интерфейсе Sentinel Monitor нажмите кнопку **"Запустить сканер"**. Система автоматически обнаружит данные с тестового стенда и занесет их в базу.
```

## Демонстрация Telegram Collector
```bash
Для проверки работы модуля мониторинга Telegram-каналов выполните следующие действия:

1. **Создание бота**:
   - Найдите в Telegram бота [@BotFather](https://t.me/botfather).
   - Создайте нового бота командой `/newbot` и получите `BOT_TOKEN`.
   - Вставьте полученный токен в файл `.env`. 

2. **Настройка канала**:
   - Создайте канал (или используйте существующий).
   - **Обязательно** добавьте вашего бота в этот канал **в качестве администратора**, иначе он не сможет читать сообщения.

3. **Конфигурация в интерфейсе**:
   - Запустите веб-интерфейс (`python -m app.web.app`).
   - Перейдите в раздел **Настройки** -> **Telegram коллектор**.
   - Нажмите "Добавить канал" и введите название ресурса и `Chat ID` канала (например, `-100...` или юзернейм `@channel_name`).

4. **Запуск**:
   - Для запуска перейдите в раздел **Настройки** -> **Telegram коллектор** .
   - Нажмите "Запустить сканер"

5. **Проверка**:
   - Отправьте в ваш Telegram-канал сообщение с тестовыми данными, например: `user@grsu.by:password123`.
   - Бот мгновенно обработает данные, и они отобразятся в "Журнале выявленных компрометаций" на вашем дашборде.

**Важное примечание:** Telegram-бот работает в режиме «прослушивания» (`event listener`). Он **не сканирует историю** сообщений канала, а реагирует только на новые сообщения, которые приходят в момент, когда скрипт уже запущен. Если вы отправите сообщение до запуска бота, он его не увидит.
```
